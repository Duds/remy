## Summary

As Dale, I want morning (and other) briefings to be composed by Remy conversationally rather than from a rigid template, so they feel natural, use Australian date format, and can prioritise what matters most.

---

## Background

Today, built-in briefings (morning, afternoon, evening) are generated by `MorningBriefingGenerator` and similar classes. They build a template string in Python and send it directly to Dale via `_send()`. **No Claude is involved.** The output is formulaic:

- Fixed structure: "Here's what you're working on:", "Make it count today. 💪", etc.
- Calendar events show ISO dates (e.g. `2026-02-27 — Alex Care`) instead of Australian format (DD/MM/YYYY or "03 Mar")
- No prioritisation or summarisation — everything is dumped in order
- Same voice every day regardless of context

User-defined automations (e.g. "Gmail quick wins") already use a different path: `run_proactive_trigger` injects a synthetic `[Reminder] {label}` message, Claude reasons and can use tools, then streams a conversational reply. That pipeline works well.

**Proposed flow:** Send briefing *data* to Remy in a concise, token-efficient format, then have Remy compose and send a conversational message to Dale. Same pattern as automations, but with structured context instead of just a label.

---

## Acceptance Criteria

1. **Structured briefing payload.** `MorningBriefingGenerator` (and optionally afternoon/evening) gains a `generate_structured()` method that returns a compact JSON-like payload: goals, calendar events, projects, downloads, birthdays, stale plans. Dates in ISO for parsing; locale hints for output.

2. **Morning briefing via Claude.** The 07:00 job invokes `run_proactive_trigger` (or a variant) with:
   - Trigger: `[Morning briefing]`
   - Context: the structured payload in the system prompt or as a synthetic user message
   - Instruction: compose a short, warm, conversational message; use Australian date format (DD/MM/YYYY) and 24-hour time; prioritise what matters most; be concise

3. **Australian date format.** All date display in briefings uses DD/MM/YYYY or "03 Mar" style. Calendar `format_event()` for all-day events is updated so raw ISO dates are not shown to the user when the payload is used for template fallback or other paths.

4. **Token efficiency.** The payload is compact: short keys, minimal redundancy. Claude receives data, not pre-formatted prose.

5. **Fallback.** If Claude fails or times out, fall back to the existing template-generated briefing so Dale still receives a message.

6. **Afternoon and evening.** Optionally apply the same pattern to afternoon focus and evening check-in (can be Phase 2).

## Implementation

**Files to modify:**

- `remy/scheduler/briefings/morning.py` — add `generate_structured()` returning `dict`; keep `generate()` for fallback
- `remy/scheduler/briefings/base.py` — add `_format_date_australian()` helper; ensure `_format_date_header()` uses Australian format
- `remy/scheduler/proactive.py` — `_morning_briefing()` calls new path: build payload → `run_proactive_trigger` with context; on failure, use `generator.generate()` + `_send()`
- `remy/bot/pipeline.py` — extend `run_proactive_trigger` to accept optional `context` dict; inject into system prompt when present
- `remy/google/calendar.py` — `format_event()` / `_parse_event_start()`: for all-day events, format date as DD/MM/YYYY or "dd MMM" instead of raw YYYY-MM-DD

**Payload shape (example):**

```json
{
  "date": "2026-03-03",
  "goals": [{"title": "Build Remy", "desc": "Personal AI assistant"}, {"title": "Clean kitchen", "desc": null}],
  "calendar": [{"time": "18:30", "title": "SMART Recovery", "url": "https://..."}, {"date": "2026-03-04", "title": "School Photos"}],
  "projects": ["folding", "AI system", "retirement"],
  "downloads": [".localized"],
  "birthdays": [{"name": "Alex", "date": "05 Mar"}],
  "stale_plans": [{"plan": "Car service", "step": "Book appointment", "days": 10}]
}
```

**System prompt addition for morning briefing:**

```
MORNING BRIEFING: You have structured context for Dale's day. Compose a short, warm, conversational message.
- Use Australian date format (DD/MM/YYYY) and 24-hour time
- Prioritise what matters most; don't list everything verbatim
- Be concise and natural — you're Remy, not a bulletin
- Include calendar highlights, top goals, and any urgent items
- Optionally mention downloads cleanup or stale plans if relevant
```

## Test Cases

| Scenario | Expected |
|----------|----------|
| Morning briefing with full context | Remy sends conversational message; Australian dates; prioritised content |
| Claude API failure | Fallback to template briefing; Dale still receives message |
| Empty goals, empty calendar | Remy composes a brief "Nothing on the radar" style message |
| Calendar with all-day events | Dates shown as "03/03/2026" or "03 Mar", not "2026-03-03" |
| Payload under 1K tokens | Context fits comfortably in system prompt |

---

## Out of Scope

- Afternoon and evening briefings (Phase 2)
- User-configurable briefing tone or length
- Changing the monthly retrospective (already uses Claude)
